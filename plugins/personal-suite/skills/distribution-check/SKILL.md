# distribution-check

何かを外部に出す前の最終チェック。送信側の防御。資料・スキル・会話ログ・スクショ等を配布/共有/公開する前に、秘密情報・個人情報・固有設定が紛れていないかを3段階で洗い出す。個人の配布物でもビジネスの配布物でも同じニーズがある。

## やること

対象(ファイル群・貼られたテキスト・会話の内容)を走査し、検出項目を**3段階のレベル**で分類して提示する。**自動で消さない** — 該当箇所(場所・引用)を示し、レベルごとの推奨アクションを添えて、持ち主が判断する。

## 3段階のレベル

### 🔴 レベル1: ほぼ確実に配布NG(秘密情報)
漏れると直接的な被害につながるもの。検出したら「除去すべき」と断定的に警告する。

- パスワード・APIキー・アクセストークン・シークレットキー
- 銀行口座番号・クレジットカード番号・各種本人確認番号
- 認証情報・秘密鍵(`.env`、`id_rsa`、`-----BEGIN ... PRIVATE KEY-----` 等)
- 検出の手掛かり: `password` `passwd` `api[_-]?key` `secret` `token` `Bearer ` `AKIA`(AWS)、長いランダム英数字列、`-----BEGIN`、口座/カード番号らしい数字列(16桁、ハイフン区切り等)

### 🟡 レベル2: おそらく配布したくない(個人情報)
本人や第三者を特定できる情報。「確認推奨」として提示する。

- 個人名(フルネーム)・家族や関係者の実名
- 住所(自宅・私的な場所)・郵便番号
- 電話番号・個人のメールアドレス
- 生年月日・年齢が特定できる記述
- 顔写真や個人を特定できる画像のパス
- 第三者の個人情報(取引先担当者名、相手の連絡先など)も含む
- 検出の手掛かり: 電話番号(`0\d{1,4}-\d{1,4}-\d{4}` 等)・メール・郵便番号(`\d{3}-\d{4}`)・住所表記(都道府県〜丁目番地)・人名

### 🟢 レベル3: 配布するとノイズになる固有設定
害は無いが、その人/組織に固有で、配布先には無意味〜混乱のもと。「汎用化を提案」する。

- 絶対パス(`/Users/(ユーザー名)/...`、社内サーバパス等)→ プレースホルダ化を提案
- 個人/社内に固有の構造名・前提(独自のフォルダ体系名、社内ツール名、独自の運用ルール)
- 個人の好み・癖の設定(口調設定、固有の判断基準、個人的なメモ)
- 内輪のコンテキスト(特定の人物・案件・過去の経緯への言及)
- 検出の手掛かり: フルパス、固有名詞の設定値、「私は〜が好き」「うちでは〜」的な個人前提

## 手順

1. 対象を特定する(配布しようとしているファイル群 / 貼られたテキスト / 共有しようとしている会話範囲)。複数ファイルなら全部走査する。
2. 上の3レベルの手掛かりで横断検索しつつ、本文も一読して文脈で拾う(機械検索だけだと固有設定=レベル3を取りこぼす)。
3. レベルごとに、**該当箇所(ファイル名・行・引用)+ なぜ該当するか + 推奨アクション**を一覧で出す。
   - 🔴 → 除去すべき(断定)
   - 🟡 → 確認推奨(出すか伏せるか本人判断)
   - 🟢 → 汎用化を提案(プレースホルダ・一般化の具体案を添える)
4. 何も無ければ「配布可能。検出ゼロ」と明言する(無理に問題を作らない)。
5. 「これは消して」「ここは汎用化して」と指示されたら、そのとき初めて修正版を作る。**承認前に書き換えない**。

## ルール
- **検出するだけ。勝手に消さない・書き換えない**(承認なしに実行しない)。
- 過剰検知でノイズにしない。レベル3は「配布先にとって無意味/混乱か」を基準に絞る(全部の固有名を挙げない)。
- 逆にレベル1は1件でも見逃さない。疑わしきはレベル1として上げる。
- 配布物が複数ファイルなら、ファイル横断で「同じ秘密が別ファイルにも」を見る。
- 判定は配布先の文脈に合わせる(社外配布か・身内共有か・完全公開かで🟡🟢の重みは変わる。不明なら配布範囲を1回尋ねる)。
