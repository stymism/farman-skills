# cal-sync

Gmail を見て、日付が確定している予定のうち **Googleカレンダーにまだ無いものだけ** を登録するスキル。受信メールから予定を拾い漏らさないための同期役。

## 前提

- Gmail と Googleカレンダーをコネクタ連携しておく(Claudeの「カスタマイズ」→「コネクタ」)。
- 必要なツール: Gmail のスレッド検索・スレッド本文取得(messageFormat=FULL_CONTENT)、Googleカレンダーの予定一覧取得(重複チェック)・予定作成。読み込まれていなければ ToolSearch で「gmail」「calendar」を検索して読み込む。

## 登録先カレンダー

**登録は必ずメインの個人カレンダー1つだけに入れる。** 共有カレンダーへの振り分けは自分が手動で行う。

理由: 「2名の予約」でも同伴者が誰かはメールから確実には判断できない(仕事の会食・友人との予定もある)。だからスキルは自動で共有カレンダーに振り分けない。まずメインに入れ、共有に出すかは自分が判断して手で移す。

**重複チェックは、自分が持っている書き込み可能なカレンダーを横断で見る**(既に共有へ移した予定を、メインに二重登録しないため)。最初の実行時に、対象にするカレンダーのIDをカレンダー一覧から確認しておく。購読系・他人のカレンダー・祝日などは照合対象から外す。

## 拾う対象(スケジュール系メール全般)

日付が確定する連絡を広く拾う。代表的な類型:

1. **フィットネス/レッスン予約** → 開始〜終了が明記された時間指定イベント。場所=スタジオ、概要欄にプログラム等。
2. **配達予定**(宅配各社・通販等) → **終日イベント**(時間帯希望なしが多いため)。概要欄に品名・送り状番号・差出人。
3. **医療・美容・飲食の予約**(歯科・クリニック・美容室・レストラン) → 時間指定イベント。場所=店名/施設名。
4. **宿泊・交通**(ホテル・鉄道・航空券) → 時間指定または複数日イベント。概要欄に予約番号・便名・チェックイン等。

判断に迷うメール(本当に予定か曖昧)は登録せず、レポートで「要確認」として挙げる。宣伝・ニュースレター・領収書のみのメールは対象外。

## 手順

### 1. 対象メールを検索する
スレッド検索で直近のスケジュール系メールを拾う。ローリング窓で探し、重複はカレンダー側で防ぐ(メールの状態は変えない)。クエリ例(複数回に分けてよい):
- `category:reservations newer_than:10d`
- `category:purchases newer_than:10d`
- `newer_than:10d (予約 OR お届け予定 OR 配達 OR 予約完了 OR ご予約 OR チェックイン OR 搭乗 OR 発車)`

ヒットしたスレッドを本文まで(FULL_CONTENT)読む。

### 2. 予定情報を抽出する
各メールから: **開始日時 / 終了日時(あれば) / 種別 / 場所 / タイトルに使う名称 / 概要欄に残す要点**。
- 日付・時刻は本文の表記から正確に取る(例: 「2026年07月01日(水) 08:15〜09:10」→ 開始 08:15 終了 09:10)。時刻が無い配達等は **終日イベント**(allDay=true)。
- タイトルは分かりやすく。例: `配達: ハンバーグ(宅配)` / `歯科 ◯◯クリニック`。
- 概要欄(description)に、メール本文から重要な部分を要約して残す(プログラム・送り状番号・品名・予約番号・店舗連絡先など)。末尾に「(cal-sync: 元メール件名)」を付ける。

### 3. 重複チェック(必須・全カレンダー横断)
登録前に、照合対象の各カレンダーについてその日(startTime=当日0:00, endTime=翌日0:00, timeZone は自分のタイムゾーン)の既存予定を取得し、**同じ予定が既にあるか**を照合する:
- 同日・同時刻帯でタイトル/場所/キーワード(店名・送り状番号・便名など)が一致 → **既登録とみなしスキップ**(どのカレンダーにあっても)。
- 判断がつかない微妙な一致は、二重登録を避けるため「要確認」に回す(重複作成より取りこぼし報告を優先)。

### 4. 登録する
未登録のものだけ、**メインの個人カレンダー**に予定を作成する:
- 時間指定: `summary` / `startTime` / `endTime` / `timeZone` / `location` / `description` / `calendarId`。
- 終日(配達等): `allDay=true`、その日付の `startTime`/`endTime`(翌日)を渡す。
- 予定を勝手に変更・削除しない。新規作成のみ。共有カレンダーへの移動は自分に任せる(スキルはやらない)。

### 5. レポート(チャット・簡潔)
```
# カレンダー同期 (YYYY-MM-DD HH:MM)
## ✅ 登録した予定 (N件)
- M/D HH:MM タイトル (場所) ← 元メール: 件名
## ⏭ スキップ (既登録 N件)
- M/D タイトル
## ❓ 要確認 (N件)
- 件名 — なぜ判断保留か(日付不明/予定か曖昧 等)
```
何も無ければ「新規の予定なし」と1行。

## ルール
- **メールの移動・ラベル付け・アーカイブはしない**(読むだけ)。
- **二重登録を絶対にしない**。迷ったら作らず「要確認」。
- 日付・時刻は推測で埋めない。読み取れないものは要確認に回す。
- 概要欄はメールの要点のみ(本文まるごとコピーしない)。個人情報の過剰転記を避ける。
- タイムゾーンは自分のタイムゾーンに固定する。
